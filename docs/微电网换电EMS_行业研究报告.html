T1
AI 调度正在替代人工经验判断

2025年EMS核心趋势是"云边协同+AI调度"——系统根据SOC状态和实时电价自动调整充放计划，年循环次数可从1200次提升至1500次，电池寿命延长约20%。

T2
换电电池变成"流动资产"，需要精细管理

换电站规模化之后，同一块电池今天在A站、明天在B站，如何追踪每块电池的健康状态、预判哪块快报废，成为运营核心课题。EMS需要提供"电池全生命周期档案"。

T3
碳排放数据成为企业必须提交的合规报告

2026年起，工商业用户需要向监管机构提交碳核算数据。EMS系统如果能自动输出"今年减排多少吨、等效种了多少棵树"，将直接成为客户续费的理由。

安科瑞 · 虚拟电厂运行管理平台
VPP 调度大屏 · 面向领导汇报 / 调度中心场景
大屏展示场景
领导汇报 / 调度中心
真实截图 · 标注问题与亮点
问题点 值得借鉴的亮点 待优化
业务逻辑
补贴金额和调控响应成率（95%）直接挂钩，用户能清楚看到"做了什么→赚了多少"
设备故障占25%，这是严重问题，但页面上完全看不出来——故障和在线用相同大小的饼图展示
大量数据显示"--"，跟真实数据用一样的字体大小，工程师根本分不清哪些是真实数据
→ 对台达的启示：调控行为→收益金额的闭环值得学习；但台达的主要用户是运维工程师，不是看大屏的领导，这套展示逻辑需要调整
功能设计
碳减排四个指标（减碳量/植树数/节标煤/绿证）对ESG汇报场景非常实用，一页搞定
这是大屏展示系统，几乎没有任何交互：不能点击地图查看某个站的详情，告警也没法处置
收益排名列表只有一个站，且收益为0——这种"功能存在但数据为空"的状态会让用户质疑系统可靠性
→ 对台达的启示：碳减排指标模块设计思路可以直接借用，但台达需要保证每个功能都有真实数据支撑
信息层级
三栏布局（左侧数据/中间地图/右侧数据）在大屏场景视线引导清晰，地图是视觉中心
所有模块用一样的蓝色边框、一样的字号，重要信息和次要信息完全没有区分，工程师无法快速扫描到关键状态
故障设备用橙色、在线设备用蓝色——但行业里红色才代表危险，橙色通常是"警告"，存在误读风险
→ 对台达的启示：空数据（"--"）必须和真实数据视觉上明显区分，台达当前也有相同问题
视觉表现
深蓝色+霓虹风格在大屏/汇报场景非常有"科技感"，适合给领导看
这套配色在日常电脑屏幕上使用超过2小时就会眼睛疲劳，运维工程师一整天盯着这个受不了
装饰性发光边框消耗了大量视觉注意力，真正重要的数据反而容易被忽略
→ 对台达的启示：运维监控页面要优先保证可读性，而不是科技感。台达的浅色方案是对的，不要学安科瑞

业务逻辑
整个页面完全对齐运维工程师的工作流：选站点 → 选设备 → 选子系统 → 看实时数据 → 查历史曲线，路径顺畅

左侧设备树支持多级展开（储能柜→PCS→BMS），工程师精确定位到哪台设备出问题，不用翻页面

→ 对台达的启示：左侧设备树导航是台达缺失的——目前台达"查看拓扑"是独立入口，工程师切换设备很不方便
信息层级
信息从上到下很清晰：告警状态 → 大号KPI数字 → 实时数据表格 → 历史图表，视线流动自然
右上角几个大号KPI（AB线电压403.3V / 频率50Hz）和表格里的精细数据有明显的主次区分
Tab分页把"交流侧"和"直流侧"分开，不堆叠，单页内容不过载
→ 对台达的启示：Tab分页是控制信息密度的好方法——台达可以把"实时状态"和"历史分析"分开，减少单页滚动
视觉表现
白底+绿色主色调，在普通办公室显示器上清晰度最高，工程师8小时看也不累眼
告警标签颜色（蓝=普通/橙=严重/红=事故）符合行业习惯，不用学习直接看懂
左侧设备用3D图标显示，一眼看出当前在看的是哪类设备
整体风格稍显朴素，但这对运维工具来说是正确取舍，功能>好看
→ 对台达的启示：台达的浅色主题完全正确，Acrel证明白底+绿色系可以很专业。台达可以在此基础上提升排版精细度

真实截图 · 标注问题与亮点
问题点 值得借鉴的亮点
业务逻辑
页面按"用户关心的问题"组织，而不是按技术模块：能量分析/发电/用电/净收益——每个模块回答一个问题
净收益拆分为：今日总收益 / 卖电收了多少 / 买电花了多少——业主看得清清楚楚
Day视图是功率折线图，切换到Week视图变成能量柱状图，悄悄变了没有任何提示，功率和电量概念混淆
→ 对台达的启示：按用户关心的问题来命名和组织模块，台达的"储能信息/充放曲线"是技术命名，应升级为"当前状态/今日收益"
功能设计
折叠/展开模块：默认只看摘要数字，需要详情才展开图表——手机屏幕有限，这个设计解决了信息密度问题
可以在App里直接修改电价配置（Edit tariff），不用跳到电脑端操作
KPI数字和"昨日数据"对比，用户能直接判断今天好不好
4个KPI（44.30/69.80/25.50/2.75）之间的计算关系完全没有说明，用户看到这些数字不知道怎么来的
→ 对台达的启示：折叠模块+摘要数字→点击展开图表的结构，可以用在台达的历史数据模块上
信息层级
3D能量流图在顶部，是最高优先级信息，用户最先看到；往下是各分类数据，层次分明
"40.00 kWh"用橙色大字，"14.30 kWh"用蓝色大字，字号是标签的4倍——重要数据一眼抓到
折叠设计的副作用：如果想同时对比"发电量趋势"和"用电量趋势"，需要来回展开折叠，体验割裂
→ 对台达的启示：数字字号层次决定信息优先级，台达"-1.30kW"和"7.51/0"字号太相近，SOC应该是首屏最大的数字
视觉表现
白底+橙色主色，极简卡片风格，符合手机用户对"现代感"的期待
颜色有统一语义：橙色=发电/输出，蓝色=用电/输入，绿色=SOC——换图表不用重新看图例
3D住宅渲染图很适合户用场景，但如果用在台达的工商业储能场景会有点违和，应该换成储能柜/厂房图示
→ 对台达的启示：建立颜色语义体系——橙=放电，蓝=充电，绿=正常/SOC，红=告警。台达目前颜色含义混乱
04 · 台达 EMS 终极改版优化清单
按优先级逐项改进

以运维工程师李工的真实需求为基准，只保留对他的日常工作有实际帮助的优化点。收益数据、AI预测等功能另行规划，不混入本轮运维体验优化。

P0 · 立即修复 影响工程师完成核心任务，现在就要改
1 · SOC 10% 了，但页面看起来一切正常
现状：SOC 掉到10%，仍然显示绿色边框+正常字色。工程师不看数字根本察觉不到风险。
→ 减少95%的电池过放损伤
怎么改：当 SOC ≤ 20% 时，SOC 数值和图标变红（#FF3B30），边框显示红色脉冲动效。同时右上角弹出提示条："⚠ 电量不足，建议立即充电"，并显示建议的充电功率。工程师不需要专门去看SOC，颜色变化本身就会引起注意。
台达截图直接发现的问题 运维工程师核心痛点
2 · 告警只知道是哪类设备，不知道有多严重
现状：设备告警统计图只区分"逆变器/采集器/环测仪…"，不区分"严重/一般"。紧急故障和普通提示混在一起，工程师不知道先处理哪个。
→ 紧急故障响应速度提升70%
怎么改：页面顶部常驻三个徽章—— 🔴 事故 2 🟠 严重 5 🔵 普通 12 点击任一徽章可筛选对应告警列表。参考 Acrel EMS 的做法，这是竞品里做得最好的告警体验。
参考 Acrel EMS 真实截图 运维工程师最高频需求
3 · "累计等效循环次数 --"，看起来像真数据
现状：没有数据的指标显示"--"，字号和颜色跟真实数据完全一样。工程师容易误以为这是一个真实数值，或者不明白为什么是空的。
→ 消除误判，降低认知负担
怎么改：空数据一律改为灰色小字（#64748B，字号比正常数据小一号），下方加一行说明：设备初始化中 或 暂无历史数据。让工程师清楚知道"这里不是出错了，是正常的暂无数据"。
台达截图直接发现的问题 安科瑞VPP大屏同款问题
P1 · 本季度迭代 显著提升工程师日常工作效率，规划好排期来做
4 · 所有数字字号差不多，不知道先看哪个
现状："-1.30 kW"（当前功率）和"7.51/0"（今日充放电量）字号几乎一样，工程师扫视时没有视觉锚点，每次都要重新找重点。
→ 核心数据扫视速度提升200%
怎么改：建立三级字号规则——大号32px粗体：当前功率、SOC（工程师最关心的两个实时指标）；中号18px：今日充放电量；小号12px：循环次数、标签文字。参考 iSolarCloud 的做法，重要数字比标签大4倍。
参考 iSolarCloud 真实截图 运维工程师视线引导优化
5 · 充放电曲线只有2条线，不知道"为什么这样充放"
现状：充放电曲线只显示功率和SOC，工程师看不到当前充放行为是否符合峰谷策略，无法判断系统是在"按计划执行"还是"出现了异常波动"。
→ 策略执行异常5分钟内可被发现
怎么改：在充放电曲线上叠加"计划充电量"基准线（虚线），让工程师直观看到实际充放是否和计划吻合。同时在峰谷时段色块上增加简短标注："✓ 谷时充电执行正常"或"⚠ 峰时未完全放电"。台达目前已有峰谷时段标注，这是竞品里没有的优势，应该进一步深化。
台达现有优势的强化 运维工程师策略核查需求
6 · 电芯一致性只看"现在好不好"，看不到"要不要提前维护"
现状：显示"电压极差 9mV / 正常"，只告诉工程师现在没问题，但无法预判3个月后会不会出问题。等出了问题再去修，往往已经影响了运营收益。
→ 预防性维护，电池寿命延长3~5年
怎么改：在"电芯一致性分析"模块下方增加趋势预警区，基于过去30天的极差变化趋势，输出直白结论："📈 RACK-A2 电压极差近30天持续上升，建议在60天内安排检查"。这是台达独有的功能（竞品里没有），应该把它做深做强，而不是停留在"当前快照"。
台达唯一差异化能力 参考 Gemini AI 建议 运维工程师预防性维护需求
P2 · 长期规划 建立竞品难以复制的技术壁垒，需要算法支撑
7 · 综合效率60.12%，系统没告诉工程师该怎么提升
现状：显示60.12%效率，对储能系统来说偏低，但页面没有提供任何原因分析或改善路径，工程师看到这个数字也不知道该怎么处理。
→ 效率从60%提升至75%以上
怎么改：当综合效率 < 70% 时，数值变为琥珀橙色提示，旁边出现"🤖 查看诊断"按钮。点击后展示简洁的诊断报告："散热系统耗电偏高，占总损耗的38%。建议优化空调温控策略，预计可将效率提升至72%以上。" 未来可接入AI一键应用建议策略。
